# Contributing

Thank you for your interest in Zero.

## Before you start

- Review the [Code of Conduct](CODE_OF_CONDUCT.md).
- Search [existing issues](https://github.com/vaibhavpandeyvpz/zero/issues) to avoid duplicates.

## Issues

- Use the bug template when reporting defects.
- Include steps to reproduce, expected vs actual behavior, and your environment (OS, Node version) when relevant.

## Pull requests

1. Fork the repository and create a branch from `main` (or the default branch).
2. Keep changes focused on one concern per PR when possible.
3. Run checks locally when applicable:

   ```bash
   npm run typecheck
   npm run build
   ```

4. Describe what changed and why in the PR description.

Maintainers will review as time permits. Feedback may be requested before merge.
